# 新 Domain 模型后台管理页面改造任务

## 1. 文档状态

- 状态：阶段一已完成，阶段二待实施
- 任务类型：跨管理端、Domain、Schema、Service、Repository、API 和 D1 的后台能力建设
- 前置基线：`doc/tasks/domain-model-rework.md`
- Domain 说明：`doc/core-domain-and-relationships.md`
- 核心商品模型：普通 Shopify 单品/套装 Product/Variant + D1 完整定制实例 + 不可变订单快照

本文记录新版 Domain 模型对应的后台信息架构、页面范围、实施阶段和验收标准。当前文档只建立执行基线，不代表页面、API 和数据表已经完成开发。

## 2. 背景与现状

当前管理端只有三个一级入口：

```text
定制模板
商品绑定
发布记录
```

现有页面可以完成 POC 模板、步骤、简单选项、尺寸字段、商品绑定和版本查询，但与新版 Domain 存在以下差距：

- 组合模板仍使用 `pieces`、`pieceSelection`、最少/最多选择数量和默认选中等自由选择语义；
- 缺少固定 `GarmentComponentDefinition` 逻辑组件编辑；
- 定制步骤和选项尚未覆盖说明、图片、展示样式、启停、排序、适用品类和稳定编码约束；
- 缺少 `OptionRelation` 管理页面；
- 尺寸仍为平铺字段，缺少 `MeasurementBlock`；
- 缺少客户量体档案、定制实例和订单快照查询页面；
- 缺少 Shopify 连接、Webhook、权限和审计等系统设置页面；
- 发布记录占用一级菜单，不利于后续信息架构扩展；
- 当前管理 UI 集中在 `app/config-admin.tsx`，继续叠加功能会增加维护成本。

## 3. 改造目标

1. 建立与新版 Domain 一致的后台信息架构。
2. 将套装编辑从自由选择单品改为固定逻辑组件及子模板映射。
3. 支持模板、步骤、选项、关系、尺寸块和尺寸字段的完整运营配置。
4. 支持商品绑定、发布校验和不可变版本查询。
5. 提供量体档案、定制实例和订单快照的受控查询能力。
6. 为 ERP/MTM 生产拆分、失败诊断和重试预留后台入口。
7. 保持 Shopify 商品、Variant、价格、销售 SKU 和库存由 Shopify 管理。
8. 将管理端拆分为可维护的页面和组件，避免单文件继续膨胀。
9. 延续已调整的 Shopify Admin 风格和顶部应用导航。

## 4. 不在本任务范围

- Shopify 商品价格、SKU 和库存编辑；
- Shopify Fixed Bundle 或 Mix-and-match Bundle 管理；
- 定制加价和 Checkout 价格覆盖；
- 在后台修改已经下单的不可变订单快照；
- 未完成身份认证时对生产开放客户量体数据；
- 完整 ERP/MTM、CRM 或门店系统实现；
- 替代 Shopify 原生商品、订单和客户后台；
- 高级经营分析和 BI 报表。

## 5. 目标信息架构

后台目标一级菜单：

```text
工作台
定制模板
商品绑定
客户量体
定制实例
订单定制
设置
```

发布记录从一级菜单移入定制模板模块：

```text
定制模板
├── 模板列表
├── 模板编辑
└── 发布记录
```

菜单继续位于应用顶部。在 Shopify Admin 嵌入环境中不再增加应用内部左侧栏，避免与 Shopify 原生导航形成双侧栏。

## 6. 页面设计

### 6.1 工作台

#### 页面目标

集中展示系统运行状态、近期业务数据和需要处理的异常。

#### 一期字段

- 模板总数、草稿数和已发布数；
- 已绑定并启用定制的商品数；
- 今日定制实例数；
- 已校验、已加购和已下单实例数；
- 订单快照成功和失败数量；
- 最近模板发布记录；
- 最近定制订单；
- 配置解析、Webhook 和 Shopify API 异常摘要。

#### 一期边界

工作台只提供基础运营状态，不实施复杂经营分析。所有统计查询必须有时间范围和索引支持，不能在请求中执行无界全表扫描。

### 6.2 定制模板

#### 6.2.1 模板列表

展示：

- 模板名称与稳定编码；
- 模板类型；
- 适用品类；
- 草稿/已发布状态；
- 当前编辑版本和最新发布版本；
- 绑定商品数；
- 更新时间。

操作：

- 搜索和筛选；
- 新建和复制；
- 编辑草稿；
- 删除未受保护模板；
- 查看绑定商品；
- 查看发布记录。

#### 6.2.2 模板基础信息

维护：

- 模板名称；
- 稳定英文编码；
- `single` 或 `composite` 类型；
- 适用品类；
- 前台按钮文字；
- 模板说明；
- 状态及并发版本。

模板编码发布后原则上不可直接修改。确需修改时必须评估商品绑定和历史订单兼容性。

#### 6.2.3 套装逻辑组件

仅组合模板显示，使用 `GarmentComponentDefinition`：

```text
code
name
category
childTemplateId
customizationEnabled
required
sortOrder
```

要求：

- 删除 `pieceSelection.min/max`；
- 删除消费者自由增删组件语义；
- 删除 `defaultSelected`；
- 上衣、西裤、马甲等组件由运营固定定义；
- 组件编码在模板内唯一；
- 子模板必须为兼容品类的单品模板；
- 必需组件必须存在于消费者配置实例中；
- 不录入 Shopify 组件 Product ID、Variant、价格或库存。

#### 6.2.4 定制步骤

维护：

- 步骤名称和稳定编码；
- 步骤类型；
- 展示样式；
- 说明；
- 是否必填；
- 是否启用；
- 排序。

允许类型：

```text
variant
options
measurements
review
```

允许展示样式：

```text
image_card
color_swatch
radio
select
text_input
```

#### 6.2.5 定制选项

维护：

- 选项名称和稳定编码；
- 说明；
- 图片 URL；
- 排序；
- 是否启用；
- 是否默认选中；
- 适用品类。

一期固定 `affectsPrice = false`，后台不提供定制加价输入。

#### 6.2.6 选项关系

新增独立页签管理 `OptionRelation`：

```text
sourceStepCode
sourceOptionCode
relationType
targetStepCode
targetOptionCode
enabled
```

支持：

```text
show
hide
enable
disable
exclude
auto_select
```

编辑器使用下拉选择步骤和选项，持久化保存稳定编码。发布前验证来源和目标存在、自动选择目标启用、无非法自引用及明显循环。

#### 6.2.7 尺寸定义

采用两级编辑：

```text
MeasurementBlock
└── MeasurementFieldDefinition
```

尺寸块维护编码、名称、说明、适用品类、启停和排序。尺寸字段维护编码、名称、说明、图片、标准单位、最小值、最大值、步长、必填、启停和排序。

后台提供 CM/IN 输入预览；D1 标准值统一保存为 CM。

#### 6.2.8 发布检查

发布前展示结构化检查结果：

- 模板、组件、步骤和选项编码唯一；
- 必填字段完整；
- 逻辑组件和子模板有效；
- 默认选项合法；
- 选项关系来源和目标存在；
- 尺寸范围和步长合法；
- 受影响商品绑定可查询；
- 当前数据通过新版 Schema。

发布成功后创建不可变版本；失败不能产生部分更新。

#### 6.2.9 发布记录

展示：

- 版本号；
- 发布时间和发布人；
- 配置摘要；
- 绑定商品；
- 与上一版本的差异；
- 只读 JSON 快照。

历史版本不可直接编辑。

### 6.3 商品绑定

#### 列表字段

- Shopify 商品标题及展示缓存；
- Product ID；
- 普通单品/普通套装类型；
- 绑定模板；
- 指定发布版本；
- `enabled`；
- Shopify 同步状态；
- 更新时间。

#### 操作

- 通过 Shopify 搜索并选择商品；
- 创建或编辑绑定；
- 指定模板和发布版本；
- 启用或停用定制；
- 解除绑定；
- 打开 Shopify 商品后台；
- 预览 Storefront 配置。

#### 保存校验

- Shopify Product 存在；
- 至少存在一个可用于加购的 Variant；
- 模板已经发布；
- 套装商品使用组合模板；
- 单品商品使用兼容品类模板；
- 组合模板包含全部必需逻辑组件。

本页面不修改 Shopify 价格、SKU 和库存。

### 6.4 客户量体

#### 列表

按 Shopify Customer、档案名称、量体对象、创建时间和更新时间查询。列表不直接展示完整尺寸。

#### 详情

展示和维护：

- Shopify Customer；
- 档案名称；
- 量体对象；
- 原始输入值和单位；
- 标准 CM 值；
- 创建和更新时间；
- 被哪些定制实例引用。

删除或修改量体档案不得影响定制实例和历史订单中的尺寸快照。

#### 安全要求

- 可靠的管理端鉴权和角色权限；
- 对象级访问控制；
- 查看和修改审计；
- 日志不记录完整尺寸；
- 未完成安全要求前不得作为生产可用页面开放。

### 6.5 定制实例

#### 列表

展示：

- 配置 ID；
- Shopify 商品和 Variant；
- 客户；
- 模板编码和版本；
- 状态；
- 逻辑组件数量；
- 创建和更新时间。

状态筛选：

```text
editing
validated
added_to_cart
ordered
```

#### 详情

展示：

- 商品与 Variant；
- 模板及版本；
- 每个逻辑组件的实际选项；
- 量体档案来源；
- 本次尺寸快照；
- 校验结果；
- 加购和下单时间；
- 关联订单；
- Line Item Properties 摘要；
- 受权限控制的原始 JSON。

`added_to_cart` 和 `ordered` 状态的实例默认只读。

### 6.6 订单定制

#### 列表

展示：

- Shopify 订单号和 Line Item；
- 配置 ID；
- 商品与 Variant；
- 客户；
- 模板版本；
- 快照状态；
- ERP/MTM 同步状态；
- 下单时间。

异常筛选包括：

- 缺少配置 ID；
- 找不到定制实例；
- 快照创建失败；
- 逻辑组件配置不完整；
- Webhook 处理失败；
- ERP/MTM 同步失败。

#### 详情

只读展示：

- Shopify Order/Line Item；
- 商品、Variant 和成交信息；
- 模板编码和版本；
- 各逻辑组件的选项和尺寸；
- 下单时展示名称；
- 快照创建时间；
- ERP/MTM 同步记录；
- 原始快照 JSON。

后续可提供导出生产配置、打印生产单和重新推送 ERP/MTM。重新推送只能发送同一不可变快照，不能修改订单配置。

### 6.7 系统设置

#### Shopify 连接

- 店铺和应用安装状态；
- API 版本；
- 权限状态；
- App Proxy 状态；
- 最近连接检查结果。

不得展示真实 Token 或 Secret。

#### Storefront 设置

- App Proxy 路径；
- 严格 CORS 允许来源；
- 默认按钮文字；
- 无配置时的前台行为；
- 主题兼容状态。

#### Webhook

- 已订阅 Topic；
- 最近接收时间；
- 成功、失败和重试数量；
- 幂等事件状态。

#### 用户与权限

建议角色：

```text
管理员
模板运营
商品运营
量体数据管理员
订单/生产人员
只读审计人员
```

#### 审计日志

记录模板修改、版本发布、商品绑定、量体档案访问、订单快照查看和 ERP/MTM 重试等操作。日志不保存完整量体数据或密钥。

## 7. 页面与 Domain/API 映射

| 页面 | 核心 Domain | 主要 API |
| --- | --- | --- |
| 工作台 | 聚合统计 View | 计划新增 `/api/admin/dashboard` |
| 定制模板 | `CustomizationTemplate` | `/api/templates/**` |
| 发布记录 | Template Version | `/api/templates/:id/versions` |
| 商品绑定 | `ProductTemplateBinding` | `/api/products/**`，后续建议更名或兼容为 bindings 资源 |
| 客户量体 | `MeasurementProfile` | 计划新增 `/api/measurement-profiles/**` |
| 定制实例 | `CustomizationInstance` | 计划新增管理端查询 API |
| 订单定制 | `OrderCustomizationSnapshot` | 计划新增管理端查询及重试 API |
| 系统设置 | Shop、Webhook、Role、Audit View | 按安全边界拆分，禁止单一万能设置接口 |

所有管理 API 必须经过可靠的 Shopify Admin 身份认证和权限检查。Route 只负责 HTTP 边界，业务规则进入 Service，SQL 进入 Repository。

## 12. 新模型切换策略

- 不兼容旧模板 JSON、`pieces` 或 `pieceSelection` 数据；
- 不保留 Legacy Schema、双读或旧数据转换代码；
- 管理端和服务端只读写 Schema v2；
- 上线前单独备份旧 D1；旧数据如需保留，由业务另行整理后按新版模型重新录入；
- Storefront API 路径保持稳定，但响应内容只输出新版组件、步骤和尺寸块结构；
- 商品绑定停用、未绑定或模板未发布仍返回 `200 + enabled:false`；
- 数据库或配置损坏继续返回可识别的 4xx/5xx；
- 主题与 Storefront 响应结构变更必须同步发布。

- 订单快照保持不可变；

## 14. 风险与控制

### 14.1 单文件管理端风险

风险：继续在 `config-admin.tsx` 中增加全部页面将导致状态、请求和视图耦合。

控制：先建立页面壳、共享类型、API Client 和领域编辑组件，再逐模块迁移。

### 14.2 旧数据切换风险

风险：旧模板数据无法被新版管理端读取。

控制：上线前备份旧 D1，并在明确的切换窗口初始化新版数据；不在运行时代码中引入兼容分支。

### 14.3 敏感数据风险

风险：量体档案和订单尺寸通过后台列表、日志或错误响应泄露。

控制：鉴权、角色权限、对象级校验、字段最小化、日志脱敏和审计。安全能力完成前不开放生产页面。

### 14.4 快照可变风险

风险：运营人员从订单页面修改配置，导致生产数据与下单内容不一致。

控制：订单快照只读；重试和导出只使用原始快照；任何人工纠错采用单独审计流程。

### 14.5 查询性能风险

风险：工作台、实例和订单列表随着数据增长产生无界查询。

控制：默认时间范围、服务端分页、必要索引和数量上限；禁止浏览器一次加载全部记录。

### 14.6 Shopify 边界混淆

风险：后台把逻辑组件错误描述为 Shopify 商品或库存单元。

控制：页面文案和字段明确区分“普通套装商品”和“逻辑组件”；不提供组件 Product ID、价格或库存输入。

## 15. 上线与回滚

- 首先在本地 D1 副本和 Shopify 开发店验证；
- 新版页面可按模块逐步启用，不要求一次开放全部菜单；
- 生产数据库迁移前完成备份并记录恢复命令；
- 新增表和字段在紧急回滚时不删除；
- 旧模板 JSON 和历史版本保留；
- 若新版模板发布失败，商品绑定继续使用上一个已发布版本；
- Storefront 和主题契约变更必须安排同一发布窗口；
- 敏感页面可通过功能开关保持关闭，直到鉴权和权限验收通过。
